## JSX란?

자바스크립트의 확장 구문이다.
JSX는 컴파일 과정을 거쳐 일반 자바스크립트 코드로 변환된다.

> **JSX의 목적**: 반복, 계산, 인라인 실행과 같은 강력한 기능을 유지하며 컴포넌트 코드를 간단하게 작성하기 위함

### 장점

- 컴포넌트 기반 아키텍처를 지원한다.
- HTML에 익숙하다면 JSX 문법은 어렵지 않다.

### 단점

- 실행을 위해서는 컴파일이 필요하다.

---

## 컴파일러

JSX를 해석하기 위해 **컴파일러**를 사용한다.

정규 표현식을 이용해 해석하지 않는데, 정규 표현식은 제대로 이해하기 어렵고 가독성 문제로 유지보수가 매우 힘들기 때문이다. 그래서 정규 표현식 대신 컴파일러를 사용해 코드를 컴파일한다.

> **컴파일러**: 특정 규칙에 따라 고급 프로그래밍 언어로 작성된 소스코드를 구문 트리로 변환하는 소프트웨어

### 컴파일러의 단계

1. **토큰화**: 문자열을 의미 있는 토큰으로 분해하는 것
2. **구문 분석**: 토큰을 가져와 트리로 변환하는 것 (문자열을 JSON 객체로 변환)
3. **코드 생성**: 추상 구문 트리(AST)에서 기계어를 생성하고, 이를 통해 생성된 기계어는 자바스크립트 엔진에 의해 실행된다.

### JIT 컴파일러

대다수의 웹 브라우저에서는 자바스크립트 코드를 효율적으로 실행하기 위해 JIT 컴파일러를 사용한다.

> **JIT 컴파일러**: 코드를 미리 변환하지 않고 실행할 때 기계어로 변환하는 방식. JVM 등에서 일반적으로 사용된다.

### 런타임

**런타임**은 엔진과 연동해 특정 환경에 맞는 기능을 더 많이 제공한다. 예를 들어, 크롬 브라우저는 엔진과 연동하는 **크로미움 런타임**을 제공한다.

## JSX로 구문 확장하기

자바스크립트 구문을 확장하려면 아래 두 작업 중 하나를 해야 한다.

1. 새로운 구문을 이해하는 엔진을 만들기
2. 엔진보다 앞서 새로운 구문을 처리하기

1번(엔진을 만드는 것)은 너무 어렵기 때문에, 당연히 2번 방식이 더 현실적이다.

### 렉서와 구문 분석기

새로운 구문을 처리하려면, 우선 JSX로 작성된 코드 문자열을 이해할 수 있는 **렉서**와 **구문 분석기**를 만들어야 한다.

> **렉서**: 토크나이저가 상태를 가지고 있고, 각 토큰이 자신의 부모나 자식에 관한 상태를 포함하고 있을 때의 토크나이저를 통칭 (토큰화 단계)

JSX를 구문 분석기를 통해 바닐라 JS로 변환한다. 이것이 바로 **Babel**, **TypeScript**, **SWC** 같은 도구가 하는 일이다 (빌드 단계).

## JSX 프라그마

JSX 프라그마는 모두 `<`로 시작하는데, 자바스크립트에서는 비교 연산을 제외하면 인식하지 못하는 문자이다. 따라서 자바스크립트 엔진은 `<`를 발견하면 오류를 던진다.

JSX는 이 `<`(JSX 프라그마)를 **함수 호출로 변환**하여 사용한다.

→ JSX 프라그마는 `React.createElement` 대신 `<`를 사용하는 **별칭**에 불과하다.

## 정리

### 1. JSX란 무엇이며, 장단점은?

JS 확장 구문. HTML과 유사한 문법으로 UI를 작성하고, 컴파일을 거쳐 `React.createElement` 호출로 변환된다.

- **장점**: 직관적인 컴포넌트 표현, HTML 친숙성, JS 기능 그대로 활용
- **단점**: 컴파일 필수 (Babel, SWC 등), 빌드 도구 설정 필요

### 2. JSX와 HTML의 차이점은?

- 속성명: camelCase (`className`, `onClick`) vs 소문자 (`class`, `onclick`)
- 태그: 모든 태그 닫아야 함 (`<img />`) vs 자체 닫힘 허용 (`<img>`)
- 표현식: `{}`로 JS 삽입 가능 vs 정적 마크업만
- 스타일: 객체 (`style={{ color: 'red' }}`) vs 문자열 (`style="color: red"`)

### 3. 텍스트 문자열은 어떻게 기계어가 되나요?

**토큰화**(문자열 → 토큰) → **구문 분석**(토큰 → AST) → **코드 생성**(AST → 기계어). 브라우저에서는 JIT 컴파일러가 실행 시점에 최적화된 기계어를 만든다.

### 4. JSX 표현식이란? 어떤 좋은 점이 있나요?

JSX 내부에서 `{}`로 JS 표현식을 삽입하는 것. 데이터에 따른 동적 렌더링이 가능하고, 별도 템플릿 언어 없이 로직과 UI를 한 곳에서 관리할 수 있다.
