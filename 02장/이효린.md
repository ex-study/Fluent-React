p.72
> JSX는 별도의 언어가 아니라 컴파일러나 트랜스파일러에 의해 일반 자바스크립트 코드로 변환되는 확장 구문입니다.
- 트랜스파일러란 무엇인가 ?
	트랜스파일러는 “한 프로그래밍 언어를 같은 수준의 다른 언어(또는 버전)로 변환해주는 도구"이다
	브라우저는 JSX를 이해하지 못하기 때문에 JS로 변환시키는 과정을 거쳐야하는데, 이를 트랜스파일링이라고 부른다

| **구분** | **설명**                               |
| ------ | ------------------------------------ |
| 컴파일러   | 고급 언어 → 기계어 (예: C → 머신 코드)           |
| 트랜스파일러 | 고급 언어 → 고급 언어 (예: JSX → JS, TS → JS) |

p. 75
> JSX 코드를 컴파일할 때 다른 문자로 바꿔 더 안전한 자바스크립트 코드가 생성됩니다.
- 왜 JSX는 더 안전하다고 말할 수 있는가?
	JSX는 문자열을 그대로 HTML에 삽입하지 않고, 자동으로 escape처리하기 때문이다
	```
	<div>{userINput}</div>
	```
	위 코드는 내부적으로 textContent처럼 동작한다. 즉, HTML로 해석되는게 아니라 문자열로 처리되기 때문이다.

p. 75
> JSX는 강력한 타이핑을 지원하므로 오류가 발생하기 전에 잡아낼 수 있습니다. (중략) 타입스크립트가 없어도 JSDoc 스타일의 주석을 사용하거나 ...(중략)
- HTML에선 JSDoc을 사용하지 못하나 ?
	여기서 말하는 "강력한 타이핑"은 JSX 자체보다는 TypeScript + JSX 조합을 의미한다.
	하지만 HTML에는 타입이라는 개념이 없다. 하지만 JSX는 JavaScript 기반이므로:
	- TypeScript 사용 가능    
	- JSDoc 사용 가능    
	- PropTypes 사용 가능
	즉, JSX는 타입 시스템을 활용할 수 있는 환경 위에서 동작한다는 점이 강점이다.

p.81
> 자바스크립트 구문을 확장하려면 새로운 구문을 이해하는 다른 엔진이 필요하거나, (중략)
- 엔진과 런타임은 무엇이 다른가 ?
	런타임은 JS가 실행되는 "환경 전체"다.

|**구분**|**역할**|
|---|---|
|JS 엔진|자바스크립트 코드를 해석하고 실행 (예: V8)|
|런타임|엔진 + 실행에 필요한 환경(API, 객체 등) 제공|
- 브라우저 런타임 → window, document
- Node.js 런타임 → global, process
- Node에는 window가 없다
